How To Write a Resume for a Job in Software Development

  1. Define your Sections

    2. There are different ways to Layout a resume, and different ways to name each section header. For these Instructions I will be using the sections in figure 1.

  2. Fill in the Name / Contact Section
    1. Your name should be at the top with a font size bigger (within reason) than the section headers.
    2. Below that, you should list your professional contact information.
      3. Figure 2

      John Something

      aJohnSomething@gmail.com (504) 555-5555 Somewheresville, SomeState, 9####

      1. Start with a more professional email address
        • Bad example: DogLover56@hotmail.com
        • Good example: aJohnSomething@gmail.com
      2. Include your phone number; some employers may call you back.
      3. Do not give all your social media accounts, they do not want to see your twitter, but a more professional one, such as LinkedIn, would be fine.
      4. If you want you can include your full address, but there are many reasons you may not want to do this. If so, you can use a more general address such as city and state.
        • In the examples John used the format of City, State, Zip.
  3. Write a Resume Objective / Resume Summary

    4. Use one of the following‚Ä¶

    1. Resume Objective

      If you are new to the workforce, or are changing careers, or need to communicate that you will/can move for the job. Then a resume objective is what you should use.

      1. In a few sentences communicate what your career goal is, and how the job you are applying for relates.
      2. For an example see Example #2
    2. Resume Summary

      A resume summary is good for when you have a work history, which you can use to communicate your experience and qualifications. For an example see Example #1

  4. Fill in Work Experience if Applicable else goto 5.

    5. If you have relevant previous work experience then you should list it here.

    Figure 3 - A Basic Template for Work History

    {Company}, {Location}

    {Dates Employed}

    {Job Title}

    • {Responsibility and Measurable Accomplishment}
    1. Start from the most recent and work backwards.
    2. First put the company name and location.
    3. Then put the dates of employment.
    4. After that, put your job title.
    5. Then in a bulleted list put the 3 most important projects and your responsibilities in them.
      • this does not have to be in chronological order, I recommend putting them in the order of most important first.
    6. Different position in same company?

      If you had a different position at the same company (promotion / transfer) then add that title followed by the dates in that position, (also add the dates in that position to the first title).

      • Remember these positions should still be in reverse chronological order.
      • (For an example and clarity see attached Example #1)
    7. Other jobs between relevant?

      If you have had a job that is not related to computer science but was for an extended period of time between CS jobs.
      Then you should list it to show you were employed during that time.

      • But don‚Äôt add the description of responsibilities and accomplishments or change in positions.
      • If more than one position just comma separate them where the job title goes.
  5. Fill in Education.

    6. This is where you mention your college degree. Do not list all the education you have ever had, just list the most important relevant ones.

  6. Fill in any Certifications if exist else goto 7.

    7. If you have any relevant certifications, you should list them here.

  7. Fill in Skills if exist else maybe you should go get some.
    1. You should list any coding languages that you are fluent in.
      • If you have a lot, try to tailor the list to be relevant to the job.
      • Try to avoid obscure, not commonly used ones unless you think they might be relevant to the job.
    2. Other relevant skills?

      If you have any other skills that would be relevant to the job, then make a subsection of Code Languages, before adding other skills.

      • Do not include ambiguous skills that are overused such as ‚ÄúHard Worker‚Äù, ‚ÄúTeam Player‚Äù etc.
    3. Make sure to list these skills with the most relevant and important first.
    4. (For an example see attached Examples #1 & #2)
  8. List Projects.
    1. If you have any projects that relate to the job you are applying for then be sure to list them
      • Try to order the list by putting the most relevant ones first.
        • By relevant I mean, what does the company your applying for do.
        • For example, if they make software for cash registers, do you have any projects that are similar, if so put them first.
      • Do not list too many as that adds clutter.
      • If you have links to these then provide them.
    2. If you have a site that shows all your projects them add that under a label of Other Projects
    3. This section is less important if you have work history.
    4. (For an example see attached Example #2)
