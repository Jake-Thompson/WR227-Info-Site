Software Development Definition

Types of Software

According to IBM there are 3 main types of software: system software, programming software, and application software. System software is software that provides core functions, this can include operating systems (OS), disk and other hardware management, etc. Programming software is software that enables the act of programming, these are tools that are necessary or helpful to the programming, debugging and other aspects of the software creation process. Application software are programs that help users with tasks, and many other things. This is the software that users primarily interact with besides the OS and websites. On phones these are normally referred to as Apps. There is a 4th type of software called embedded software which is software that controls devices that are not computers and smartphones. These devices might be stand-alone or part of the Internet of Things (IoT) [1].

Types of Software Development

There are primarily three job titles that engage in software development (SD). The first type normally thought of are programmers, also known as coders. This job primarily writes source code to perform specific tasks by interpreting instructions from software developers/engineers [1]. This is the Job that is most likely to be automated by Artificial Intelligence (AI). Generative AI such as ChatGPT can already do this to a degree.

Software engineers design not only software but systems to solve problems. They do this using engineering principles and the scientific method. They must coordinate with the mechanical and electrical engineers to make sure that everything works together [1]. This Job may require certification in some states [3].

Software developers on the other hand, fit right in between these two jobs, less formal than software engineers, but have more responsibility than programmers. They can write source code at times and also be in charge of the entire software development lifecycle (see more about life cycle here) [1].

Software Developer vs. Software Engineer

The difference between a software developer and a software engineer can be confusing. Some companies might even interchange the titles or have their own meaning for them. The main difference between a software developer and engineer is focus.

A software developer focuses more on the user experience and the design that that entails. This does not mean they only work on the user interface, but everything that could impact the user experience. A lot of the time they work directly with clients to find out what they want and present designs for approval before moving forward with the development process [1].

Software engineers on the other hand create software to perform specific tasks and functions. These could be used by many different applications. They do this rigorously using the scientific method [1].

Resume

Resume Summary vs. Resume Objective

A resume summary summarizes your job experience highlighting your qualities and skills. It’s a way to quickly present how you can be an asset [5]. A resume objective on the other hand highlights your short-term career goals, while explaining why you are looking for employment. The main reason you would use a resume objective over a summary is if you are new to the job market or new to this career [6].

Work History Formatting

The most normal way to format work history is to list your jobs in reverse chronological order (most recent first). You start with the company’s name and location. The location does not need to be the full address. And on the next line the dates you worked at that company overall. Followed each title you held at that company with a list of important responsibilities and accomplishments [7].
